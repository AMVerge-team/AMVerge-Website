# Export Profiles

An export profile bundles all the settings AMVerge uses when you click **Export
Now**: workflow, container, codec, audio, and hardware. You can keep several
profiles and switch between them.

## Active profile

The **Active Profile** is the one Export Now uses. Newly created profiles become
selectable from the export panel dropdown. Each profile has a **Profile Name**
and a **Profile Icon** shown in that selector.

## Workflow

| Workflow | What it does |
|---|---|
| **Export video (stream copy / remux)** | Copies the original stream, no re-encode. Fastest, lossless. |
| **Export video (re-encode)** | Re-encodes with the chosen codec. Needed when changing codec/quality. |

## Container

The file wrapper for the output:

| Container | Notes |
|---|---|
| **MP4** | Most compatible, default |
| **MKV** | Flexible, good for many tracks |
| **MOV** | For ProRes / editor workflows |

## Codec (re-encode only)

| Family | Profiles |
|---|---|
| **H.264 / AVC** | Main, High, High 10, High 4:2:2 |
| **H.265 / HEVC** | Main, Main 10, Main 12, Main 4:2:2 10 |
| **ProRes** | 422 LT, 422, 422 HQ, 4444, 4444 XQ |

## Audio

| Option | Notes |
|---|---|
| **Keep audio copy** | Original audio, untouched |
| **AAC 192 / 320 kbps** | Compressed, compatible |
| **PCM 16 / 24-bit** | Uncompressed |
| **FLAC / ALAC** | Lossless compressed |
| **Opus 160 / MP3 320** | Compressed |
| **No audio** | Strip audio |

## Hardware

| Mode | Behavior |
|---|---|
| **Auto GPU / CPU** | Use the GPU when available, fall back to CPU |
| **GPU** | Force GPU encoding |
| **CPU** | Force CPU encoding |

GPU encoding (NVIDIA) is much faster for H.264/H.265 re-encodes. ProRes and some
profiles always use the CPU.
